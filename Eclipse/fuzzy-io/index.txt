<h1>FuzzyIO</h1>
<p>FuzzyIO is a server for generating "fuzzy" resolution real estate development scenarios. </p>
<p>This is just the "back end". If you want to use fuzzy builder,  you need to use a "front end" user interface like <a href="http://teasquare.ddns.net">openSUI</a>.</p>

<h2>Model Components</h2>
<p>A <b>Function</b> describes the principal activity within a volume of enclosed space. (e.g. "Residential" or "Retail")</p>
<p>A <b>Voxel</b> is a small, indivisible unit of enclosed space consisting of a square base, height, and single <b>Function</b></p>
<p>A <b>Zone</b> is a grid-based array of <b>Voxels</b> with a common <b>Function</b>. If a <b>Zone</b> consists of multiple floors, all floors share the same footprint.</p>
<p>A <b>Volume</b> is a stack of <b>Zones</b> that all share a common footprint, but each zone may have a different <b>Function</b> and number of floors</p>
<p>A <b>Podium Volume</b> is a special <b>Volume</b> with a footprint defined by an off-set from a <b>Parcel</b>'s edges and subtracting any <b>Podium Exclusion Areas</b></p>
<p>A <b>Podium Exclusion Area</b> is an area, defined by a polygon, where <b>Voxels</b> cannot be placed.</p>
<p>A <b>Tower Volume</b> is a special <b>Volume</b> with a rectangular footprint. It must be placed within a <b>Parcel</b>.</p>
<p>A <b>Parcel</b> is the set of flat, grid-based <b>Voxels</b> that fit within the boundaries of a defined polygon. Parcels cannot overlap each other.</p>
<p>A <b>Scenario</b> is a specific configuration of <b>Voxels</b> specified by <b>Parcels</b>, <b>Volumes</b>, and <b>Zones</b>.</p>
